# 本地 Markdown WYSIWYG 编辑器

一个纯前端、无后端、零依赖的本地 Markdown 所见即所得编辑器，使用浏览器 File System Access API 直接读写本地文件。

## ✨ 核心特性

- **纯前端实现**：HTML + JavaScript + CSS，无任何第三方依赖
- **本地文件访问**：使用浏览器原生 File System Access API
- **自动保存**：编辑内容后自动保存，1秒防抖
- **永不格式化源码**：只做最小修改，不全局重排、不压扁引用、不乱加空行
- **所见即所得**：支持基本的 Markdown 语法渲染
- **新建笔记**：一键创建新的 Markdown 文件

## 🚀 快速开始

### 1. 浏览器要求

需要支持 **File System Access API** 的现代浏览器：
- Chrome 86+
- Edge 86+
- Opera 72+

> ⚠️ 注意：Safari 和 Firefox 目前不支持此 API

### 2. 使用方法

1. **直接打开** `index.html` 文件
   - 双击打开，或在浏览器中打开该文件

2. **授权访问文件夹**
   - 点击左侧「📂 选择 notes 文件夹」按钮
   - 在文件选择器中选择本项目的根目录（包含 notes 文件夹的目录）
   - 浏览器会请求权限，点击「允许」

3. **开始使用**
   - 左侧会显示 notes 文件夹中所有 `.md` 文件
   - 点击文件名即可打开编辑
   - 点击「+ 新建」创建新笔记

## 🎯 核心设计原则

### 永不格式化源码

这是本编辑器最重要的设计原则：

1. **只做最小修改** - 只写入用户实际编辑的内容
2. **保留原始格式** - 完整保留换行、空格、缩进
3. **不全局重排** - 从不重新格式化整个文档
4. **不压扁引用** - 保持引用嵌套结构完整
5. **不乱加空行** - 不自动添加或删除空行

实现方式：使用 `innerText` 而非 `innerHTML` 读写内容，最大程度保留用户的原始输入。

## 🔧 技术说明

### File System Access API

本编辑器使用浏览器原生的 File System Access API：
- `window.showDirectoryPicker()` - 选择目录
- `FileSystemDirectoryHandle` - 目录操作
- `FileSystemFileHandle` - 文件操作
- `createWritable()` - 写入文件内容

### 安全说明

- 所有文件操作都在浏览器沙箱内进行
- 需要用户显式授权才能访问本地文件
- 编辑器本身不会上传任何数据到服务器
- 所有数据都保存在你的本地 notes 文件夹中

## ❓ 常见问题

**Q: 为什么打开后看不到我的文件？**
> A: 首次使用需要点击「选择 notes 文件夹」按钮授权访问。选择包含 notes 文件夹的项目根目录。

**Q: 为什么浏览器提示"不安全"？**
> A: File System Access API 需要 HTTPS 或 localhost 环境。直接打开本地文件（file:// 协议）在部分浏览器中可能受限。建议使用本地服务器运行，如 `python -m http.server`。

**Q: 可以在其他电脑上使用吗？**
> A: 可以，只要复制整个项目文件夹到其他电脑，用支持的浏览器打开即可。

**Q: 会丢失我的笔记格式吗？**
> A: 不会。本编辑器严格遵循"永不格式化源码"原则，最大程度保留原始格式。
